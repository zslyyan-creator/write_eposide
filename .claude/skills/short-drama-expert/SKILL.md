# 短剧爆款创作专家

## 核心理念

**爆款铁三角**：对抗 + 变化 + 共情

**商业化铁律**：
- 黄金3秒必须有钩子
- 每90秒一个小高潮
- 每集结尾必有卡点
- 付费点前必须有足够的情绪积累

---

## 五步创作流程

> ⚠️ **重要规则**：
> 1. 必须完成当前步骤并获得用户确认后，才能进入下一步
> 2. 每步输出必须严格遵循 [TEMPLATES.md](TEMPLATES.md) 中定义的格式
> 3. 确认后立即将内容写入对应文件

---

### 第一步：标题与核心创意

**目标**：确定项目基本信息

**输出格式**：严格按照 [TEMPLATES.md](TEMPLATES.md) 中的「项目信息模板」

**验证清单**：
- [ ] 标题有"网感"（反差/悬念/身份标签/数字）
- [ ] 一句话卖点清晰（谁+困境+行动+反转）
- [ ] 核心爽点明确
- [ ] 目标受众定位准确

**确认后操作**：
1. 创建项目文件夹
2. 写入 `00_项目信息.md`

⏸️ **确认点**：请用户确认后继续

---

### 第二步：故事简介与梗概

**目标**：完成投流简介 + 四幕梗概

**输出格式**：严格按照 [TEMPLATES.md](TEMPLATES.md) 中的「故事梗概模板」

**梗概要求**：
- 简介：100字内，用于投流
- 四幕结构：建置(10%) → 对抗升级(40%) → 真相抉择(40%) → 高潮解决(10%)
- 必须标注每幕的情绪走向

详见 [STRUCTURE.md](STRUCTURE.md) 四幕二十节拍详解

**确认后操作**：写入 `01_故事梗概.md`

⏸️ **确认点**：请用户确认后继续

---

### 第三步：人物小传

**目标**：为每个重要角色创建独立角色卡

**输出格式**：严格按照 [TEMPLATES.md](TEMPLATES.md) 中的「角色卡模板」

**角色分类**：
| 类型 | 文件命名 | 字数要求 |
|---|---|---|
| 主角 | `主角_姓名.md` | 500字 |
| 对手 | `对手_姓名.md` | 300字 |
| 配角 | `配角_姓名.md` | 150字 |

**必填字段**：
- 基本信息（姓名/年龄/职业/外貌）
- 标志性特征（动作/口头禅）
- 性格与缺陷
- 欲望（Want vs Need）
- 人物弧光

详见 [CHARACTERS.md](CHARACTERS.md) 人物设计指南

**确认后操作**：
1. 创建 `02_角色卡/` 文件夹
2. 为每个角色写入独立文件

⏸️ **确认点**：请用户确认后继续

---

### 第四步：分集大纲

**目标**：逐集拆解，每集独立文件，包含爽点分析和付费点验证

**输出格式**：严格按照 [TEMPLATES.md](TEMPLATES.md) 中的「分集大纲模板」

**单集大纲必须包含**：

```markdown
## 基本信息
- 集数/总时长/在四幕中的位置

## 本集核心
- 核心事件（一句话）
- 核心冲突（A vs B）
- 本集爽点类型

## 场景拆解
- 场景1：开场钩子
- 场景2-N：冲突升级
- 最后场景：卡点设计

## 情绪曲线
- 开场情绪 → 中段情绪 → 结尾情绪

## 卡点设计
- 卡点类型（悬念/情绪/反转）
- 具体设计
- 下集承接方式

## 付费点检查（如果是付费集）
- 前置爽点是否足够
- 情绪积累是否到位
- 悬念强度评估
```

**付费点验证**（关键！）：
在设计付费点集数时，必须回答：
1. 观众此时的情绪状态是什么？（愤怒/期待/揪心/上头）
2. 前面铺垫了哪些爽点让观众"上头"？
3. 这个卡点的悬念强度是否足以驱动付费？
4. 如果观众不付费会错过什么关键信息？

详见 [HOOKS.md](HOOKS.md) 卡点设计与付费点策略

**确认后操作**：
1. 创建 `03_分集大纲/` 文件夹
2. 每集写入独立文件：`第XX集_大纲.md`
3. 更新 `05_付费点分析.md`

⏸️ **确认点**：请用户确认后继续

---

### 第五步：剧本正文

**目标**：逐集输出可拍摄剧本

**前置条件**（必须满足）：
在写任何一集剧本前，必须先读取：
1. `00_项目信息.md` - 了解项目定位
2. `01_故事梗概.md` - 了解整体故事
3. `02_角色卡/` 下所有文件 - 了解所有角色
4. 本集之前所有 `04_剧本正文/第XX集_剧本.md` - 了解前情
5. `03_分集大纲/第XX集_大纲.md` - 本集大纲

**输出格式**：严格按照 [TEMPLATES.md](TEMPLATES.md) 中的「剧本正文模板」

**剧本格式规范**：

```markdown
# 第X集

## 前情提要
（自动生成：上一集结尾的关键信息）

## 本集概要
（从大纲复制：核心事件、冲突、爽点）

---

X-1 景：地点，日/夜，内/外
人物：XXX、XXX

△环境描写（一句话）
△人物动作/状态

**角色A**（语气）：台词。

△动作反应

**角色B**（语气）：台词。

**角色A**OS：内心独白。

---

X-2 景：...

---

【本集卡点】
△关键画面定格描写

—— 本集完 ——

## 下集预告
（一句话悬念）
```

**符号系统**：
| 符号 | 用途 | 示例 |
|---|---|---|
| △ | 动作/神态描写 | △林青宴冷笑，放下茶杯。 |
| **人名** | 台词前加粗 | **林青宴**（冷笑）：哦？是吗。 |
| （语气） | 表情/动作提示 | （咬牙）、（故作镇定） |
| OS | 内心独白 | **林青宴**OS：这笔账我记下了。 |
| VO | 画外音 | **小欢**VO：老大，目标到了。 |
| 【闪回】 | 回忆片段 | 【闪回】...【闪回结束】 |

详见 [FORMAT.md](FORMAT.md) 完整格式规范
详见 [METHODOLOGY.md](METHODOLOGY.md) 七维度创作方法论

**写作铁律**：
- ❌ 禁止心理描写（不可拍）→ ✅ 用△动作/微表情外化
- ❌ 禁止连接词 → ✅ 短句、断句、口语化
- ❌ 禁止自我介绍式台词 → ✅ 通过冲突自然暴露
- ❌ 禁止无意义过场 → ✅ 每句话推进剧情

**确认后操作**：
1. 创建 `04_剧本正文/` 文件夹
2. 每集写入独立文件：`第XX集_剧本.md`

## 类型速查

| 类型 | 核心爽点 | 经典套路 |
|---|---|---|
| 甜宠 | 双向奔赴、反差宠溺 | 契约婚姻、先婚后爱 |
| 复仇 | 打脸、逆袭、真相大白 | 重生复仇、隐藏身份 |
| 战神 | 身份碾压、实力打脸 | 退役归来、隐藏大佬 |
| 赘婿 | 扮猪吃虎、身份反转 | 隐忍三年、一朝觉醒 |
| 穿越 | 金手指、改变命运 | 带系统、重生逆袭 |
| 虐恋 | 错过、误会、破镜重圆 | 双向救赎、先虐后甜 |

详见 [FORMULA.md](FORMULA.md) 类型公式大全
